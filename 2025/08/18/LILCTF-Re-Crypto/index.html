ez_math

先分析是怎么加密的

先把flag括号里的内容分为两个大整数lambda1和lambda2

生成一个512位的素数p作为有限域的模数，生成两个128位素数的二维向量v1和v2

矩阵A由v1和v2作为行向量构成，矩阵B由v1乘以lambda1和v2乘以lambda2构成

C = A⁻¹ × B，因为B是A的每行分别乘以不同λ值得到的，这个运算实际上会得到一个对角矩阵，对角线元素就是lambda1和lambda2

但是C并不是对角矩阵，求解特征值，矩阵 C的特征值 λ 满足：λ2−tr(C)λ+det(C)=0

tr(C) = C[0][0] + C[1][1]（迹）
det(C) = C[0][0] * C[1][1] - C[0][1] * C[1][0]（行列式）

mid_math

分析题目给的加密过程

  1. 参数生成：
    • 生成一个64位的素数 p，并定义在 GF(p)下的运算
    • 生成一个随机的密钥 key，范围在 2^62 到 p之间
    • 生成五个64位的素数向量 v1, v2, v3, v4, v5
    • 生成四个64位的素数 a, b, c, d，以及 e = 0
  2. 矩阵构造：
    • 构造矩阵 A，其行向量为 v1, v2, v3, v4, v5
    • 构造矩阵 B，其行向量为 v1*a, v2*b, v3*c, v4*d, v5*e
    • 计算 C = A.inverse() * B
    • 计算 D = C ** key
  3. 加密：
    • 将 key转换为字节并填充到16字节，作为AES的密钥
    • 使用AES ECB模式加密填充后的 flag

所以现在我们要恢复key,然后解密msg

C = A.inverse() * B,它的特征值就是{a,b,c,d,0} ,D=C^key，所以它的特征值是 {a^key,b^key,c^key,d^key,0}

eigens = C.eigenspaces_right() 对 C 做右特征分解，得到每个特征值 λ 及其对应的特征空间。这些 λ 应该就是 a,b,c,d,0,然后利用这些特征向量去作用在 D 上

Dv = D * v mu = Dv[i] / v[i]

这样得到的 μ 就是 D 在同一个特征向量上的特征值,配对出 (λ, μ) = (λ, λ^key)

解μ≡λ^key(modp)，找到两种方法：

1.以 λ 为底直接做离散对数key≡logλ(μ)(modn)，拿到多对（λi,μi）就得到多条同余
key≡ki ( mod  ni)，再用 CRT合并，找出符合区间的key值

2.换到群的一个生成元 g 上,得到的是一条弱一些的同余（模数是 (p−1)/d）。对多对 (λ,μ)得到多条同余，用广义 CRT合并,找出符合区间的key值
