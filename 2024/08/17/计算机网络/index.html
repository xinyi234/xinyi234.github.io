计算机网络

计算机网络

概论

1.1 什么是internet

节点：主机及其上运行的应用程序 路由器、交换机等网络交换设备

边：通信链路

接入网链路：主机连接到互联网的链路

主干链路：路由器间的链路

互联网是TCP/IP一簇协议为主的网络

互联网是分布式的应用进程及为应用进程提供通信服务的基础设施，基础设施包括主机应用层以下的协议实体

协议定义了在两个或多个通信实体之间交换的报文格式和次序，以及在报文传输或接收或其他事件方面采取的动作

主机=端系统

Internet标准：RFC， IETF

1.2网络边缘

网络边缘：主机 应用程序

网络核心：互连的路由器，网络中的网络

接入网：有线或无线通信链路

网络边缘接入网络核心

客户/服务器模式：客户端向服务器请求、接收服务。问题：可扩展性

对等模式：很少专门的服务器，每个节点既是客户端也是服务器，如分布式文件分发系统

面向连接的通信方式：在数据传输之前做好准备，只有端系统知道

可靠性：不重复，不丢失，不失序

TCP协议是有连接的，可靠的，具有拥塞控制的特性

UDP协议是无连接的，不可靠，无流量控制，无拥塞控制

1.3网络核心

网络核心：路由器的网状网络

电路交换：

  • 带宽被分成片（频分、时分、波分）

  • 信令系统为两者的通信分配一条独享的线路

  • 独享资源：不同享

  • 每个呼叫一旦建立就能保证性能

  • 如果没有数据发送，被分配的资源会浪费

频分：带宽比较宽，按频率频分多路复用

时分：通讯能力按时间分

波分：光纤通信，可用波段分为若干个

电路交换用于电话，不适合计算机通信：

  • 连接建立时间长
  • 计算机之间的通信具有突发性，浪费片
  • 可靠性不高

分组交换

  • 网络带宽资源不再分为一个个片，传输时使用全部带宽
  • 主机之间传输的数据被分为一个个分组
  • 先存储后转发
  • 延迟比电路交换多
  • 共享性

排队和延迟：如果到达速率>链路的输出速率：

  • 分组将会排队，等待输出
  • 如果路由器缓存用完了，分组将会被抛弃

网络核心的关键功能：

路由：决定分组采用的源到目标的路径

转发：将分组从路由器的输入链路转移到输出链路

分组交换：统计多路复用 ，允许更多用户使用网络

分组交换按照有无网络层的连接，分成：

  1. 数据报网络
    • 分组的目标地址决定下一跳
    • 在不同的阶段，路由可以改变
    • 无须建立连接，有数据就传输
    • 无状态路由器，不维护主机的通信状态
  2. 虚电路网络
    • 建立虚拟电路，带有标识
    • 建立网络层连接
    • 存储转发

1.4接入网与物理媒体

边缘网络通过接入网接入网络核心

接入方式：

  • modem：利用电话线接入互联网，将上网数据调制加载音频信号上，在电话线上传输，在局端将其中的数据解调出来，不能同时上网和打电话
  • dsl:调制和解调，同样利用现存的电话线，DSL线路上的数据被传到互联网，语音被传到电话网，可以同时打电话和上网
  • 线缆网络：有线电视信号线缆双向改造 FDM：在不同频段传输不同信道的数据，数字电视和上网数据。
    • 非对称：最高30Mbps的下行传输速率，2Mbps上行传输速率
    • 线缆和光纤网络将家庭用户接入到ISP 路由器
    • 各用户共享到线缆头端的接入网络
  • 电缆模式
  • 企业接入网络：端系统经常直接接到以太网络交换机上 10Mbps,100Mbps1Gbps
  • 无线接入网络：各无线端系统共享无线接入网络（通过基站或接入点）
    • 无线LANs：建筑物内部，802.11b/g(WiFi):11，54Mbps传输速率
    • 广域无线接入：由电信运营商提供，1~10Mbps,4G 5G

物理媒体：

  • 在传输-接收对间传输比特
  • 物理链路：在每个传输-接收对，跨越一种物理媒体
  • 导引型媒体：信号沿着固体媒介被导引：同轴电缆、光纤、双绞线
  • 非导引型媒体：信号自由传播：无线电
  • 无线链路：开放空间传输电磁波。类型有地面微波、LAN、广域、卫星

1.5Internet结构和ISP

互联网服务提供商ISP

互联网内容提供商ICP

端系统通过接入ISPs连接到互联网

接入ISPs相应的必须是互联的，任何两个端系统可互相发送分组到对方，导致网络的网络非常复杂

将每两个ISPs直接连接，不可扩展

竞争：如果全局ISP是有利可为的业务，则会有竞争者

合作：通过ISP之间的合作可以完成业务的扩展，肯定会有对等互联的结算关系

业务会细分为全球接入和区域接入。

ICP接入互联网要向ISP交费，然后ICP可能会构建自己的网络，将它们的服务、内容更加靠近端用户，向用户提供更好的服务，减少自己的运营支出

Internet结构：松散的层次模型

中心：第一层ISP ，国家/国际覆盖，速率极高

直接与其他第一层ISP相连，与大量第二层ISP和其他客户网络相连

ISP之间的连接：

  • POP：高层ISP面向客户网络的接入点，如一个低层ISP接入多个高层ISP，涉及费用结算
  • 对等接入：2个ISP对等互联
  • IXP：多个对等ISP互联之处
  • ICP：自己部署专用网络，同时与各级ISP连接

1.6 分组延时、丢失和吞吐量

分组丢失和延时是怎么发生的

在路由器缓冲区的分组队列，分组到达链路的速率超过链路输出的能力，分组等待拍到队头被传输，如果队列溢出，则分组被丢弃。

四种分组延时：

  1. 节点处理延时

    检查错误，查路由表

  2. 排队延时

  3. 传输延时

    将分组发送到链路的时间=L/R，存储转发延时

  4. 传播延时

    物理链路的长度/传播速度

从一个节点到另一个节点为一跳(hop)

处理延时通常是微秒数量级或更少

排队延时取决于拥塞程度

传输延时对低速率的链路很大（如拨号），通常为微秒级

传输延时几微秒到几百毫秒

排队延时取决于流量强度 I=La/R

分组丢失的原因：

  • 链路的队列缓冲区容量有限
  • 当分组到达满的队列是，该分组将会丢失
  • 丢失的分组可能会被前一个节点或源端系统重传，或根本不重传

1.7 协议层次和服务模型

分层思想解决复杂的问题，每一层次通过接口向上层提供服务，对等层的模块通过协议交换信息

协议是对等的水平关系，协议通过接口借助下层提供的服务才能实现，协议的目的是为上层提供更好的服务

服务：低层实体向上层实体提供他们之间的通信的能力

服务用户：上层的应用 服务提供者：下层的协议

服务访问点：上层使用下层提供的服务通过层间的接口

原语：提供服务的形式（如函数调用）

服务类型：

  • 面向连接的服务

  • 无连接的服务

    两个对等层实体在通信前不需要建立连接，不预留资源，不可靠

服务和协议的区别：

  • 服务：在系统内部通过层间接口使用原语下层向上层提供服务
  • 协议：对等曾实体在相互通信的过程中需要遵守的规则

各层次的协议数据单元：

  • 应用层：报文
  • 传输层：报文段：TCP段、UDP数据段
  • 网络层：分组（无连接方式：数据报）
  • 数据链路层：帧
  • 物理层：位

1.8 历史

1960以前，线路交换

1961-1972，早期的分组交换概念 ARPAnet IMP:接口报文处理机 1972年ARPAnet有15个节点，第一个email程序出现

1972-1980：专用网络和网络互联

1990，2000‘s：商业化，Web，新的应用

应用层

2.1 应用层原理

网络应用的体系架构：

  • 客户-服务器模式(C/S)

    资源在服务器，客户端向服务器请求资源，可扩展性差，可靠性差

  • 对等模式

    每个节点既请求其他节点的服务，也为其他提供服务，管理困难

  • 混合体

    Napster音乐分发系统 即时通讯

进程通信

​ 进程：在主机运行的应用程序，在同一主机内，使用进程间通信机制通信（操作系统定义），不同主机，通过交换报文来通信

客户端进程：发起通信的进程

服务器进程：等待连接的进程

应用层通过socket api接口使用传输层提供的服务交换报文

标识应用进程三要素：主机ip, tcp还是udp, 端口号

web应用默认端口80，ftp默认端口21，telenet默认端口23

问题2： 层间接口必须要携带的信息：

  • 要传的报文（对本层来说，SDU)
  • 谁传的：应用进程的标识：IP+TCP(UDP)端口
  • 传给谁：对方的IP+TCP(UDP)端口

传输层实体（tcp或udp实体）根据这些信息进行tcp报文段(udp报文段)封装

  • 源端口号，目标端口号，数据等
  • 将IP地址往下交IP实体，用于封装IP数据报：源IP，目标IP

如何减少层间传输的信息量：

​ TCP socket:两个进程之间的通信需要之前要建立连接，且通信会持续一段时间，通信关系稳定，可以用一个整数表示两个应用实体之间的通信关系，代表源IP，源端口，目标IP，目标端口四元组，是一个本地标识

UDP socket:两个进程之间的通信无需建立连接，每个报文都是独立传输的，前后报文可能给不同的分布式进程，只能用一个整数表示本应用实体的标识，表示本IP，本端口，但是传输报文时，必须提供对方的IP，端口，接收报文时，传输层需要上传对方的IP，端口

socket是应用层和传输层的约定，代表会话关系

应用层协议：定义了运行在不同端系统上的应用进程如何交换报文

应用协议仅仅是应用的一个组成部分

  • 公开协议：由RFC文档定义，允许互操作，如HTTP，SMTP
  • 私有协议：协议不公开，如Skype

传输层向应用层提供的服务的指标：

  • 数据丢失率
  • 延迟
  • 吞吐量
  • 安全性

2.2 Web and HTTP

Web页：由一些对象组成

对象可以是html文件，jpeg图像，java小程序，声音剪辑文件

通过url对每个对象进行引用

http:超文本传输协议，web的应用层协议，客户/服务器模式

http跑在tcp之上。客户发起一个与服务器的tcp连接，端口号为80，服务器接受客户的tcp连接，在浏览器与服务器交换报文，连接关闭。无状态服务器。

http/1.0 非持久连接 http/1.1持久连接

请求方式：非流水线方式，流水线方式

http请求报文：ascii可读

提交表单输入：

  • post方式：网页通常包括表单输入，包含在实体主体中的输入被提交到服务器
  • url方式：get方式，输入通过请求行的url字段上载

cookies维护客户端-服务端状态

4个组成部分：

  1. 在HTTP响应报文中有一个cookie的首部行
  2. 在HTTP 请求报文含有一个cookie的首部行
  3. 用户端系统图保留一个cookie文件，由用户的浏览器管理
  4. web站点有一个后端数据库

web缓存（代理服务器）：用户设置浏览器通过缓存访问web，浏览器将所有http请求发给缓存，如果对象在缓存，直接返回对象，如果不在，缓存请求原始服务器，再返回对象
